# 指令相关问题

### 超时的回调

```java
//断连后或者设备故障，发送指令超时，cmd是超时的指令
 @Override
    public void timeOut(String cmd) {

    }
```

### step1

如果是sdk对接过程中，出现了指令超时，首先需要排查是否指令冲突，一般同一时间只能调用一个指令，两个指令尽量间隔在300ms，上传历史数据的时候，如果数据在上传期间，进行了主动测量，一般会提示“繁忙”，所以尽量页面上做好提示

### step2

如果不能确定指令超时是代码问题还是固件问题，可以使用上位机来测试，排除掉sdk的干扰，如果上位机发送指令没有返回结果，大概率是不支持该指令，或者固件有问题，可以联系我们进行排查。

### step3

如果上位机发送指令有回调，但是超过了5s(新的sdk已改为8s)才有回应，这个在sdk里认为超时，可以反馈给我们

### 指令没有响应

如果发送了指令，没有回调，或者没有响应，如何排查

### step1

连接上戒指，如果立马发送指令，设备可能没准备好，会有超时，建议延时3s以后发送指令，并且不能同时发送多个指令，建议链式发送，每个指令之间相隔300ms以上

### step2

如果发送指令，只有发送的日志，没有收到的日志，可能是标志位没设置对，android需要查看一下，是否在连接成功的回调里，设置了`BLEUtils.setGetToken(true);`
